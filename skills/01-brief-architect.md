# Brief Architect

Use this when you have a rough product idea and want Claude Design to turn it into a clear, build-ready direction.

## Input checklist

- Product or page type
- Target audience
- Main user problem
- Primary conversion goal
- Tone or brand direction
- Constraints: device, deadline, complexity, style

## Output you want

- one-sentence product summary
- audience and pain points
- page or app structure
- key UI blocks
- recommended visual direction
- copy priorities
- things to avoid

## Paste into Claude Design

```text
Act as a product strategist and interface lead.

I want you to turn this rough idea into a sharp design brief for Claude Design.

Product/Page:
[describe the product, feature, landing page, or concept]

Audience:
[who this is for]

Main problem:
[what pain this solves]

Primary goal:
[signup / purchase / book demo / install / trust / explain]

Tone:
[minimal / premium / playful / technical / editorial / bold]

Constraints:
[mobile-first / desktop-first / one-page / fast to build / premium look / simple UX]

Deliver the result in this format:
1. Core idea
2. Audience and jobs-to-be-done
3. Information hierarchy
4. UI sections or screens
5. Recommended visual direction
6. Copy priorities
7. Risks and what to avoid

Be concrete. Do not stay high level. Make decisions.
```
